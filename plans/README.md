## Findings surviving verification but NOT planned this round

These were confirmed/adjusted by the audit but left unplanned — lower leverage,
larger/looser scope, or already partly mitigated. Recorded so they aren't lost.

- **RPC client `send()` lacks backpressure handling** (`packages/rpc-client/src/rpc-client.ts:681`): write() return value ignored, no `drain` listener. Downgraded to MED — a `stdin` error handler + 30s timeout already reject pending requests, so the practical failure is a slow-fail, not a silent hang. Worth doing if long-running sessions show hung requests.
- **`MessageBatcher.enqueue()` after `destroy()` drops silently** (`packages/daemon/src/message-batcher.ts:115`): add a debug log on the destroyed-drop. Trivial; fold into plan 015 if convenient.
- **Web API route handlers lack unit tests** (~43 `route.ts` files): error paths are covered only via integration suites. Real gap; a `web/app/api/__tests__/` layer is a medium effort better scoped as its own initiative.
- **Malformed-RPC-input tests missing** (`packages/rpc-client`): golden fixtures exist, adversarial/truncated-JSONL inputs untested. The protocol does wrap parse errors; add a `rpc-client-malformed.test.ts` when touching that area.
- **`test:unit` flakes under process isolation**: the CI green/red signal is not fully trustworthy under parallel `--experimental-test-isolation=process` contention (heavy files pass standalone). Fixing the verification baseline (split heavy suites into a sequential job / concurrency cap) is high-value but a CI-infra effort of its own.
- **Dashboard polls `/api/visualizer` every 30s without change detection** (`web/components/gsd/dashboard.tsx:144`): a 10s server-side cache already blunts the subprocess cost; adding ETag/`304` is a clean follow-up, not urgent.
- **Three doc systems drift** (`docs/`, `gitbook/`, `mintlify-docs/`): `docs/user-docs/` is canonical (most active); gitbook/mintlify may be stale. Auditing and consolidating is an L-effort docs initiative.
- **Two near-identical RPC client copies** (`packages/rpc-client` vs `packages/gsd-agent-modes/src/modes/rpc`): genuine duplication (it caused the 1000-vs-5000ms shutdown drift plan 012 fixes at both sites). De-duping is worthwhile but a deliberate refactor — surface to maintainers before doing it, given VISION's "no cosmetic refactors" rule.

## Findings considered and rejected

- **Task/slice closed→open guard missing from Status Transition Core**: explicitly deferred by ADR-030 until sanctioned `reopenTaskStatus`/`reopenSliceStatus` faces exist; not planned independently.
- **Log redaction regex misses lowercase secret key names**: the regex already carries the `i` flag, so lowercase keys are matched; no change needed.
- **Static tool-contract gate sites not folded into one helper**: the four sites already share `getUnitWorkflowDispatchReadinessError` in `tool-contract.ts`; further unification is not a clean, bounded change.
- **"God modules" (`auto-prompts.ts`, `auto.ts`, `auto-post-unit.ts` 4000+ LOC)**: verifier found these are intentional cohesive API layers, heavily tested via `ForTest` seams (31–35 test files each), not untested landmines. VISION.md explicitly bans refactor-for-its-own-sake. Rejected.
- **`paths.ts` triple-cache "complexity"**: the three caches with manual invalidation are an intentional design validated by `stale-dirlistcache-4648.test.ts`. Rejected.
- **"657 untested source files / 87% uncovered"**: false — the cited high-churn files are covered through exported test seams and 30+ test files each. Rejected.
- **Path-resolution / status-classification "duplication"**: the resolver functions already delegate to common helpers (`resolveFile`, `resolveMilestoneFile`); evidence was mis-located. Rejected.
- **"Silent catch blocks on critical paths"**: cited line ranges were systematically wrong (pointed at interface definitions); the actual catches carry the house-style justification comments. Rejected.
- **`recharts` unconditionally bundled**: `web/components/ui/chart.tsx` is not imported anywhere in `web/`, so recharts is not in any shipped bundle. If anything, delete the unused `chart.tsx`; not a live perf cost. Rejected as a perf finding.
- **Wrong pending-verification field cleared in `orchestrator.ts`**: verifier confirmed the two fields (`pendingVerificationRetry` vs `pendingVerificationRetryDispatch`) are intentionally distinct and each is cleared correctly; a passing test asserts the behavior. Rejected.
- **Detached GitHub milestone sync unawaited**: labeled "best-effort detached sync" and covered by `recovery-finalize-logs.test.ts`; by design. Rejected.
- **Pre-merge-check shell-injection**: the tokenizer feeds `execFileSync(binary, args)` with no `shell:true`, so shell metacharacters are inert. Rejected.
- **Per-message WebSocket auth / Hono CORS / Hono path-traversal advisories**: the gateway uses raw `http.createServer` (no Hono CORS/serve-static middleware reachable), and the WS connection is bearer-authenticated at connect over WSS; the specific exploit paths are unreachable. Rejected (undici DoS via discord.js is real and IS planned — see 016).
- **Stale-render detection tests skipped / flat-phase migration "incomplete"**: `detectStaleRenders` is intentionally disabled during the shipping v1.4.0 flat-phase migration (documented design), not an accidental coverage hole. Tracked as direction (finish the migration), not a bug.
